오늘의 과제

문제를 푸는 차원에서는 딱히 없습니다
그동안 모델, 동작 인스턴싱, 배경까지 한 번에 압축해서 진도를 빼시느라 고생하셨습니다

○ 복습
1. [ 실행 ] 오늘은 만들어진 게임씬을 수행(캡쳐) -> 여러분 나름대로 응용해보세요
2. [ 모션 ] 모델을 바꾸거나, 나루토.FBX에서 빠졌던 점프, 펀치 같은 동작을 넣어보거나
3. [ NPC ] 새로운 적을 넣어보거나, 상관없는 NPC (여우라든가)가 뛰어다니거나
4. [ 배경 ] 새로운 맵, 혹은 밤 하늘...제 3의 리소스를 추가해보거나

중요한 건 오늘까지 진행한 흐름을 이해해주시는 겁니다
나중에 스스로 나만의 게임을 만들 수 있도록 혹은 프레임워크가 다른 환경으로 바뀌어도 최초의 메인cpp등에서 필요한 기능을 추적해서 찾을 수 있도록

○ 앞으로 진행할 부분은 우선 이렇습니다
모두 하면 제일 좋은데 시간이 무한하지 못해서 우선순위를 둘 수 있음

1. 눈에 보이는 시각 효과 강화 (버퍼 응용, 파티클, 그림자, 특수효과 등)
	└ 여기부터 진도를 다시 나갈 겁니다
2. 길 찾기 알고리즘 (2진 탐새겡 의한 의사 결정 = A*)
3. 공간 분해 알고리즘 (노드 분할에 의한 데이터 구분 알고지름 -> 길 찾기나 데이터 압축에 사용한다)
	└ [ 키워드 ] 바이트리(2개), 쿼드리(4개), 옥트리(8개) 이 중에서 게임에서는 주로 쿼드트리를 사용합니다

이중에 다 못하는게 있으면 실습 시간에 가볍게라도 설명하거나, 엔진 가서라도 진행합니다
